Basic Types:

Pointless has four basic types: numbers, strings, booleans, and labels.

Numbers

All numbers in Pointless use double-precision floating-point representation. Pointless supports a number of common mathematical operations.

-- some numbers a = 12 b = 4.56 -- numerical operations: -- addition -- subtraction -- multiplication -- division -- modulus -- exponentiation nums = [a + b, a - b, a * b, a / b, a % b, a ** b] -- inequality operations: -- less-than -- greater-than -- less-than or equal-to -- greater-than or equal-to bools = [a < b, a > b, a <= b, a >= b]

Strings

In Pointless, strings are the fundamental type for representing pieces of text - Pointless has no character type. Two strings can be concatenated into a new string using the '+' operator.

-- an empty string a = "" -- an non-empty string b = "I lost the game" -- string addition c = b + ", again!"

Booleans

Booleans in Pointless are represented with the keywords 'true' and 'false'.

-- the booleans a = true b = false -- boolean operations: bools = [a and b, a or b, not a]

Labels

Labels in Pointless are like symbols in Lisp - they can be used much like enumeration values in other language, except that the enumeration doesn't have to be declared explicitly. Label names start with a capital latter, and can contain alphanumeric characters. Two labels can be compared with the 'is' operator.

-- some labels a = Foo b = Bar -- label comparison: results = [a is Foo, b is Foo, Foo is a]

Conditionals:

Pointless has two conditional constructs 'if' and 'switch', which are used to select the value of one of several expression based on boolean conditions.

If

If conditionals follow the form 'if' condition 'then' 'then expression' else 'else expression'. If conditionals can be broken into multiple lines for readability. If expressions can be nested, as in the second example below, where the else of the first conditional is a second conditional. The expressions in the 'then' and 'else' branches are not evaluated unless the branch is selected by the condition.

-- a simple conditional getSign(n) = if n >= 0 then Positive else Negative -- a nested conditional getSignZero(n) = if n > 0 then Positive else if n < 0 then Negative else Zero output = rangeStep(-3, 3, 1) |> map(getSignZero) |> println

Switch

Switch conditionals consist of one or more cases of the form 'case' condition result. When a switch is evaluated, the conditions are evaluated in-order; when a case is found with a true condition, the switch conditional returns the value of the result expression for the case. A switch conditional can also include a default case of the form 'otherwise' result (must come last in the list of cases). The value of the default case will used if no case conditions are true. If no conditions are satisfied and the switch has no default case, the conditional will raise an error.

As in if conditionals, the result expressions of cases are evaluated lazily.

In the example below, the first switch returns a label based on a number; the second switch returns a string based on a label value.

-- a switch statement with a default case getSignZero(n) = switch { case n > 0 Positive case n < 0 Negative otherwise Zero } -- a switch statement with no default getSignSymbol(sign) = switch { case sign is Positive "+" case sign is Negative "-" case sign is Zero "0" } output = getSignZero(-4) |> getSignSymbol |> println

Definitions:

As seen in the previous section, conditionals in Pointless are themselves expression - they evaluate to a value. A Pointless program is made up entirely of expressions and definitions - Pointless does not have statements, or even an explicit execution flow, unlike imperative programming languages. Furthermore, every expression in a program must be part of a definition - Pointless doesn't allow functions to be called without using the resulting value as other languages do.

Variables

Variable definitions take the form variableName '=' expression. A variable name can only be used once in a given scope.

n = 1024

Functions

Function definitions take the form functionName(param1, param2, param3) = functionBody (for any number of parameters >= 1). Parameters must be valid variable names (alpha-numeric, beginning with a lower-case letter).

double(n) = n * 2

Functions may be defined recursively, without any special annotation, as in the factorial example below.

factorial(n) = if n == 0 then 1 else n * factorial(n - 1)

Writing programs in Pointless is often easiest when functionality is divided among many small functions. The function definition syntax is intended to make the conversion between variable definitions and functions simple, as in the example below, where a definition for a variable called average

average = (a + b) / 2

is converted to a function, simply by adding a list of parameters to the left-hand side of the definition.

average(a, b) = (a + b) / 2

Where

A where clause can be used to create new scopes and define local variables. These clauses take the form expression 'where' definition(s). A where clause with a single definition (an intermediate value used in the function) is shown below.

volume(radius, height) = base * height where base = pi * radius ** 2

Alternatively, a where clause can contain a bracketed group of multiple definitions.

distance(x1, y1, x2, y2) = sqrt(dx ** 2 + dy ** 2) where { dx = x1 - x2 dy = y1 - y2 sqrt(n) = n ** .5 }

Calling Functions:

mul(a, b) = a * b output = println(mul(2, 3))

Partial Application

mul(a, b) = a * b triple = mul(3) output = println(triple(2))

Pipe Operator

output = println(range(3)) output = range(3) |> println output = println(map(mul(2), range(3))) output = range(3) |> map(mul(2)) |> println

Data Structures:

Lists

-- a list of numbers numbers = [1, 2, 3] a = head(numbers) -- 1 b = tail(numbers) -- [2, 3] -- empty list empty = [] a = [] ++ [] -- [] b = [1] ++ [2, 3] -- [1, 2, 3] c = [false] ++ ["asdf"] ++ [] -- [false, "asdf"] -- end seq once 1 is reached -- otherwise current value is next in the output sequence, -- along with the sequence starting at the next value collatz(n) = if n == 1 then [1] else [n] ++ collatz(next(n)) next(n) = if n % 2 == 0 then n / 2 else n * 3 + 1 take(n, list) = switch { case n == 0 [] case list is Empty [] -- take one value, and reduce the number -- of values left to take by one otherwise [head(list)] ++ take(n - 1, tail(list)) }

Arrays

-- an array of numbers a = [1 2 3] x = a[0] -- 1 (arrays are zero-indexed) -- a two-dimensional array of numbers b = [-1 0 1] [-2 0 2] [-1 0 1] y = a[1][2] -- 2 (row 1, column 2)

Sets

-- an set of labels directions = {North, East} e = toSet([]) -- an empty set x = North in directions -- true y = South in directions -- false

Maps

-- an map of from strings to numbers values = {"pawn": 1, "knight": 3, "bishop": 3, "rook": 5, "queen": 9} -- an empty map e = {} x = "pawn" in values -- true y = "king" in values -- false

Tuples

-- an unlabeled tuple point = (3, 4) -- destructuring assignment (x, y) = point -- a tree structure made with labeled tuples tree = Branch(Leaf(1), Branch(Leaf(2), Leaf(3))) x = tree is Branch -- true y = tree is Leaf -- false -- making a tuple by calling a label leaf = Leaf l = leaf(0) -- Leaf(0)

Objects

-- an object pet = { type = Dog age = 4 name = "rufus" } output = [pet.name, pet.type, pet.age] |> format("{} ({}, age {})") |> println

Comprehensions:

For

squares = for n in range(10) yield n ** 2 pairs = for a in range(3) for b in range(3) yield (a, b)

When

pairs = for a in range(3) for b in range(3) when a > b yield (a, b) for var in iter body -- is equivalent to concatMap(var => body, iter) when cond body -- is equivalent to if cond then body else [] yield body -- is equivalent to [body]

Laziness:

Lists

repeat(value) = [value] ++ repeat(value) output = repeat("How long till we get there?") |> enumerate |> printLines range(n) = iterate(add(1), 0) |> take(n)

Defs

a = 1 output = [a, b, c] |> println b = 2 c = 3 x = {myself = x; value = 123} output = println(x.myself.myself.value)

Performance:

Tail-Call Optimization

sum(list) = if list is Empty then 0 else head(list) + sum(tail(list)) output = sum(range(1000)) |> println sumHelper(result, list) = if list is Empty then result else sumHelper(head(list) + result, tail(list)) sum = sumHelper(0) output = sum(range(1000)) |> println

Concatenation

repeat(value) = [value] ++ repeat(value)
