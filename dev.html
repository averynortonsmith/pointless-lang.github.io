Development

Remaining technical issues:

  • gc is slow

    • Heap becomes fragmented over time, leading to slowdown

  • gc is untested

    • Needs to be tested

  • vm stack traces are could be improved

    • Tail-call optimization currently leads to poor stack-traces

  • vm code is poorly structured

    • VM implementation is spread across header files - need to find a way to make the vm modular without sacrificing performance / compiler optimizations (mainly inlining)

  • compiler doesn't optimize bytecode

    • Many opportunities to improve performance here

  • vm is slow

    • Could be optimized in many ways - but need benchmarks first

  • io interface is limited

    • Currently can only read from / write to stdin / stdout

  • visible mutation

    • Updating maps, sets, and arrays results in mutations that are visible to the user - accessing a stale reference to an updated object should result in a runtime error

  • better installtion instructions

    • Installation process could be made more robust / automated. Windows installation should not rely on the WSL.
