Getting Started:

See the install page for installation instructions.

Hello World!

The following code implements the classic "Hello World!" program in Pointless. The program can be run by executing the Pointless binary with a source containing the following code as an argument.

output = println("Hello World!")

Make source file:

echo 'output = println("Hello World!")' > hello.ptls
./pointless hello.ptls

Comments

Comments in Pointless begin with two dashes '--' and span the rest of the line. Comments can be placed on the same line as code (following the code), or on their own line, as seen below. A long bar of dashes can be used to separate blocks of code and provide documentation for functions.

----------------------------------------------------------- -- check whether a number is even - returns true for even -- ints, false for odd ints or non-int numbers isEven(n) = n % 2 == 0 -- even ints equal zero mod 2

Basic Types:

Pointless has four basic types: numbers, strings, booleans, and labels.

Numbers

All numbers in Pointless use double-precision floating-point representation. Pointless supports a number of common mathematical operations.

-- some numbers a = 12 b = 4.56 -- numerical operations: -- addition -- subtraction -- multiplication -- division -- modulus -- exponentiation nums = [a + b, a - b, a * b, a / b, a % b, a ** b] -- inequality operations: -- less-than -- greater-than -- less-than or equal-to -- greater-than or equal-to bools = [a < b, a > b, a <= b, a >= b]

Strings

In Pointless, strings are the fundamental type for representing pieces of text - Pointless has no character type. Two strings can be concatenated into a new string using the '+' operator.

-- an empty string a = "" -- an non-empty string b = "I lost the game" -- string addition c = b + ", again!"

Booleans

Booleans in Pointless are represented with the keywords 'true' and 'false'.

-- the booleans a = true b = false -- boolean operations: bools = [a and b, a or b, not a]

Labels

Labels in Pointless are like symbols in Lisp - they can be used much like enumeration values in other language, except that the enumeration doesn't have to be declared explicitly. Label names start with a capital latter, and can contain alphanumeric characters. Two labels can be compared with the 'is' operator.

-- some labels a = Foo b = Bar -- label comparison: results = [a is Foo, b is Foo, Foo is a]

Conditionals:

Pointless has two conditional constructs 'if' and 'switch', which are used to select the value of one of several expression based on boolean conditions.

If

If conditionals follow the form 'if' condition 'then' 'then expression' else 'else expression'. If conditionals can be broken into multiple lines for readability. If expressions can be nested, as in the second example below, where the else of the first conditional is a second conditional. The expressions in the 'then' and 'else' branches are not evaluated unless the branch is selected by the condition.

-- a simple conditional getSign(n) = if n >= 0 then Positive else Negative -- a nested conditional getSignZero(n) = if n > 0 then Positive else if n < 0 then Negative else Zero output = rangeStep(-3, 3, 1) |> map(getSignZero) |> println

Switch

Switch conditionals consist of one or more cases of the form 'case' condition result. When a switch is evaluated, the conditions are evaluated in-order; when a case is found with a true condition, the switch conditional returns the value of the result expression for the case. A switch conditional can also include a default case of the form 'otherwise' result (must come last in the list of cases). The value of the default case will used if no case conditions are true. If no conditions are satisfied and the switch has no default case, the conditional will raise an error.

As in if conditionals, the result expressions of cases are evaluated lazily.

In the example below, the first switch returns a label based on a number; the second switch returns a string based on a label value.

-- a switch statement with a default case getSignZero(n) = switch { case n > 0 Positive case n < 0 Negative otherwise Zero } -- a switch statement with no default getSignSymbol(sign) = switch { case sign is Positive "+" case sign is Negative "-" case sign is Zero "0" } output = getSignZero(-4) |> getSignSymbol |> println

Definitions:

As seen in the previous section, conditionals in Pointless are themselves expression - they evaluate to a value. A Pointless program is made up entirely of expressions and definitions - Pointless does not have explicit execution flow, unlike imperative programming languages. Furthermore, every expression in a program must be part of a definition - Pointless doesn't allow functions to be called without using the resulting value as other languages do.

Add definitions occur in a scope. Constructs like functions, where clauses, and objects create new local scopes for their parameters / definitions. Local scopes form closures over non-local variables.

Variables

Variable definitions take the form variableName '=' expression. A variable name can only be used once in a given scope.

n = 1024

Functions

Function definitions take the form functionName(param1, param2, param3) = functionBody (for any number of parameters >= 1). Parameters must be valid variable names (alpha-numeric, beginning with a lower-case letter).

double(n) = n * 2

Functions may be defined recursively, without any special annotation, as in the factorial example below.

factorial(n) = if n == 0 then 1 else n * factorial(n - 1)

Writing programs in Pointless is often easiest when functionality is divided among many small functions. The function definition syntax is intended to make the conversion between variable definitions and functions simple, as in the example below, where a definition for a variable called average:

average = (a + b) / 2

is converted to a function, simply by adding a list of parameters to the left-hand side of the definition:

average(a, b) = (a + b) / 2

Where

A where clause can be used to create new scopes and define local variables. These clauses take the form expression 'where' definition(s). A where clause with a single definition (an intermediate value used in the function) is shown below.

volume(radius, height) = base * height where base = pi * radius ** 2

Alternatively, a where clause can contain a bracketed group of multiple definitions.

distance(x1, y1, x2, y2) = sqrt(dx ** 2 + dy ** 2) where { dx = x1 - x2 dy = y1 - y2 sqrt(n) = n ** .5 }

Calling Functions:

Functions call syntax uses parentheses around group of comma-separated argument expressions. Calling a function with more arguments than it has parameters results in an error.

mul(a, b) = a * b output = println(mul(2, 3))

Partial Application

When a function is called with fewer arguments than its number of parameters, a new function object is created which will accept the remaining function parameters, while storing the already-supplied argument values. In the example below, triple is a function which takes one parameter (b). When triple is called, it returns value of the original function mul called with the stored value for param a (3) and the new value for b (2).

mul(a, b) = a * b triple = mul(3) output = println(triple(2))

Pipe Operator

Function calls can also be expressed using the function-pipe operator '|>', where 'a |> b' is equivalent to 'b(a)', or 'a |> b |> c' is equivalent to 'c(b(a))'. Thus, the following call:

output = println(range(3))

could instead be expressed as like this:

output = range(3) |> println

The function-pipe operator can be combined with partial application to express a group of nested function calls like this:

output = println(map(mul(2), range(3)))

as a linear sequence of transformations, which is more in-line with the logical flow of the program itself:

output = range(3) |> map(mul(2)) |> println

Data Structures:

Pointless has a number of built-in data-structures - that is, types which contain other types within them. These data-structure types are: lists, arrays, sets, maps, tuples, and objects.

Lists

A list is used to store an iterable sequence of values. Unlike arrays, in which elements can be accessed through numerical indecies, list elements can only be accessed through iteration. Lists are created through the syntax below: a square-bracket enclosed comma-separated sequence of values. An empty list is represented with empty brackets '[]', or equivalently with the label 'Empty'. The function 'head' returns the first value in a list, the function 'tail' returns a list of all values after the first. Calling head or tail on an empty list results in an error.

-- a list of numbers numbers = [1, 2, 3] a = head(numbers) -- 1 b = tail(numbers) -- [2, 3] -- empty list empty = []

Two lists can be concatenated with the concat operator '++' to form a new list. Note that lists can also contain different types of values at once.

a = [] ++ [] -- [] b = [1] ++ [2, "three"] -- [1, 2, "three"] c = [false] ++ ["asdf"] ++ [] -- [false, "asdf"]

The concat operator can be used in recursive functions to build lists procedurally. The 'collatz' function below creates a list containing a collatz sequence starting at a given number.

-- end sequence once 1 is reached -- otherwise current value is the start of the output sequence, -- along with the sequence starting at the next value collatz(n) = if n == 1 then [1] else [n] ++ collatz(next(n)) next(n) = if n % 2 == 0 then n / 2 else n * 3 + 1

In addition to producing lists, functions can take lists as input and process them recursively, like the 'take' function below, which takes the first 'n' elements from an input list (or fewer if the input list contains fewer than 'n' elements).

take(n, list) = switch { case n == 0 [] case list is Empty [] -- take one value, and reduce the number -- of values left to take by one otherwise [head(list)] ++ take(n - 1, tail(list)) }

Arrays

Arrays, like lists, contain sequence of elements, represented by square-bracked enclosed space-separated sequence of values. Unlike lists, arrays have a fixed size (that is, they cannot be concatenated). Their elements are accessed through the index syntax below. Pointless also includes a special syntax for nested two-dimensional arrays, also seen below.

-- an array of numbers a = [1 2 3] x = a[0] -- 1 (arrays are zero-indexed) -- a two-dimensional array of numbers b = [1 0 1] [2 0 2] [1 0 1] y = a[1][2] -- 2 (row 1, column 2)

Sets

A set is an unordered collection of unique values. Uniqueness is determined according the language's equality rules. The operator 'in' can be used to see if a set contains a given value.

-- an set of labels directions = {North, East} e = toSet([]) -- an empty set x = North in directions -- true y = South in directions -- false

Maps

A map is an unordered collection of key / value pairs, where each key is unique. As with sets, uniqueness follows equality rules. The operator 'in' is used to check whether the keys of a map contain a given key value.

-- an map of from strings to numbers values = {"pawn": 1, "knight": 3, "bishop": 3, "rook": 5, "queen": 9} -- an empty map e = {} x = "pawn" in values -- true y = "king" in values -- false

Tuples

A tuple is a ordered group of values. Unlike a list or array, the elements in a tuple normally serve different purposes, and may be different types. Tuples may be labeled or unlabeled - labeled tuples are formed by calling a label value with one or more arguments - this results in a tuple of the arguments with the given label. The values in a tuple are accessed through destructuring assignment, as shown below. As with plain labels, the labels of tuples can be checked using the 'is' operator.

-- an unlabeled tuple point = (3, 4) -- destructuring assignment (x, y) = point -- a tree structure made with labeled tuples tree = Branch(Leaf(1), Branch(Leaf(2), Leaf(3))) x = tree is Branch -- true y = tree is Leaf -- false -- making a tuple by calling a label stored in a variable leaf = Leaf l = leaf(0) -- Leaf(0)

Objects

An object is a set of definitions - as in where clauses, these can be variable definitions or function definitions. The fields (definitions) within an object can be accessed through using the syntax 'objectValue.fieldName' show below.

-- an object pet = { type = Dog age = 4 name = "rufus" } output = [pet.name, pet.type, pet.age] |> format("{} ({}, age {})") |> println

Comprehensions:

List comprehension syntax provides another way to create lists in Pointless. A list comprehension takes the form 'for' variable 'in' values body, where the body of the comprehension is either a yield statement or another comprehension. The yield statement marks the expression used to generate the elements in the resulting list (the yield keyword is necessary to disambiguate nested comprehensions). To generate the new list, the given variable name is assigned to each value in the values list, and the body expression is evaluated to create new elements in the output list.

For

 -- starting with a list of numbers 0 through 9, generate -- a new list containing the squares of these numbers squares = for n in range(10) yield n ** 2 -- generate a list of pairs -- [(0, 0), (0, 1), (0, 2), (1, 0), (1, 1) ...] pairs = for a in range(3) for b in range(3) yield (a, b)

When

Comprehensions can also include when statements of the form 'when' condition body, where the body expression is evaluated only when the condition is met - otherwise no elements are returned for that iteration of the comprehension. When statements can be used to filter the output from a comprehensions.

-- generate a list of pairs -- [(0, 0), (1, 0), (1, 1), (2, 0), (2, 1), (2, 2)] pairs = for a in range(3) for b in range(3) when a >= b yield (a, b)

List comprehensions are syntactic sugar - the precise semantics of for, when and yield statements can be understood by looking at canonical forms that the compiler translates these statements to.

for var in iter body -- is translated to concatMap(var => body, iter) when cond body -- is translated to if cond then body else [] yield body -- is translated to [body]

Laziness:

Expressions in Pointless are normally evaluated eagerly. There are exceptions to this rule, like the branches of conditional statements, as described previously. There are two other important instances where the language introduces laziness: lists and definitions.

Lists

Lists introduce laziness in two ways: first, the elements in a list expression are not evaluated until the elements themselves are accessed (using the head function). Second, when two lists are concatenated, the right-hand value in the concatenation is not evaluated until it is accessed (when traversing the list using the tail function). As a result of this, functions with recursive calls on the right-hand side of a concatenation expression can be used to generate infinite lists, as seen in the example below, which continuously generates new list elements and prints them to the console.

repeat(value) = [value] ++ repeat(value) output = repeat("How long till we get there?") |> enumerate |> printLines

The Pointless library contains many functions that can process infinite lists - for example the take function, which can be used to make finite list from an infinite list.

range(n) = iterate(add(1), 0) |> take(n)

Note that list comprehensions can also be used to process and generate lazy lists.

Defs

Like lists, the values of definitions are also calculated lazily. Specifically, when a new set of definitions is encountered (in the global scope, a where clause, an object, etc), all definitions are registered in the new scope before any are evaluated. As a result, the order of definitions in a program does not matter, as shown in the example below.

a = 1 output = [a, b, c] |> println b = 2 c = 3

Note that unlike other definitions, arguments to functions are calculated eagerly.

Normally, accessing the value of a circularly-defined variable will result in a call-stack overflow. However, lazy definitions allow some types of recursive structures to be defined - for example recursive objects in the global scope. While it is possible to use these recursive structures in some situations (as shown below), using them with other operations, like 'print', will make the program enter in infinite loop.

x = {myself = x; value = 123} output = println(x.myself.myself.value)

Performance:

Notes on two important performance considerations when writing programs in Pointless.

Tail-Call Optimization

Pointless performs tail-call optimizations during program execution - that is, a function which returns the result of another function call does not extend the call-stack when performing this inner call. The following code, for example, is not in tail-call form and cannot be optimized, since the addition step is performed after the inner call returns. This function would create a call-stack overflow if called on a large list.

sum(list) = if list is Empty then 0 else head(list) + sum(tail(list)) output = sum(range(1000)) |> println

On the other hand, the following function performs the addition step before making the recursive call, and will thus be optimized, making it suitable for input lists of arbitrary length.

sumHelper(result, list) = if list is Empty then result else sumHelper(head(list) + result, tail(list)) sum = sumHelper(0) output = sum(range(1000)) |> println

Concatenation

During concatenation, Pointless makes a copy of the left-hand list being concatenated. As a result, the concatenation operation has a time complexity that is linear in the length of the left-hand list. This also means that the concatenation operation only works when the left-hand list has a finite length.

On the other hand, the right-hand side list in a concatenation remains lazily evaluated. This means that concatenation operations with a recursive call on the right-hand side are still in tail-recursive form, and can be used to generate infinite, lazy lists, as shown in the repeat function below, which creates an infinite list of a given value, repeated.

repeat(value) = [value] ++ repeat(value)

Equality:

Notes on the different ways to compare values in Pointless.

Basic Types

Basic types (numbers, booleans, strings, and labels) can be compared using the '==' operator (note that comparing non-integer numbers using '==' may fail due to floating-point inaccuracies). Additionally, tuples can be compared using the '==' operator if all of their member values are comparable using '=='.

123 == 123 -- true 123 != 4 -- true

Data Structures

For performance reasons, comparison of other data structures (lists, arrays, sets, maps, objects, and tuples not fitting the criteria above) using the '==' operators compares these objects according to their location in memory. As a result, comparison of two data-structures will return false even if those structures contain the same values, unless they are two references to the same original structure. Comparison of functions follows the same rule.

To overcome this limitation, Pointless defines two functions, 'equals' and 'notEquals', which can be used to perform deep-comparison of values containing data-structures.

{0} == {0} -- false equals({0}, {0}) -- true

Note that comparison of two values of different types using '==' or 'equals' will always return false.

Label Comparison

As seen previously, the 'is' operator can be used to compare labels, or to check the label of a labeled tuple. Additionally, 'is' can be used to determine the type of other values in pointless. For example, the expression value 'is' PtlsNumber can be used to check if a value is a number, value 'is' PtlsBoolean can check booleans, and so on. Note that value 'is' PtlsLabel, and value 'is' PtlsTuple will also return true for any labels and tuples, respectively. All of the example expressions below will return true.

123 is PtlsNumber -- number "asdf" is PtlsString -- string true is PtlsBool -- boolean Foo is PtlsLabel -- label Foo is Foo -- label sum is PtlsFunc -- function {}.!getDelKey is PtlsBuiltIn -- built-in {1} is PtlsSet -- set {"one": 1} is PtlsMap -- map ["a", "b"] is PtlsList -- list [0 0 0] is PtlsArray -- array {x = 0; y = 2} is PtlsObject -- object Bar(1, 2) is Bar -- labeled tuple Bar(1, 2) is PtlsTuple -- labeled tuple (1, 2) is PtlsTuple -- unlabeled tuple

Updates:

In Pointless, arrays, sets, maps, and objects are mutable types - that is, their values can be updated. Pointless provides several ways to perform updates on data-structures, depending on their types. Currently, these update operations work by mutating the structure being updated, and returning a reference to the structure. Because of this, aliasing of structures is still visible to the user - however, future work on the language may remove this visible aliasing.

Via With

With statements can be used to update arrays, maps, and objects. These statements take the form value 'with' updateDefinitions. In an update definition, the character '$' is used to represent the structure being updated. An update definition assigns a new value to some index or field on the original structure. In the below example, the array values has its first element (zero index) set to the value zero. The value returned by the with statement is the value of the updated structure.

values = [1 2 3] newVals = values with $[0] = 0 -- [0 2 3]

With statements can contain multiple update definitions, as in the example below. As with where clauses, with statements with multiple definitions must wrap these definitions in brackets. The below with statement returns a reference to the values array with the elements at indecies zero and one swapped. Note that the right hand sides of the update definitions are calculate before the updates are processed - which allows the swap to occur without using a temporary variable.

values = [4 10] -- newVals = [10, 4] newVals = values with { $[0] = values[1] $[1] = values[0] }

The following example demonstrates the nested update of an object containing a map and an object in its fields.

structure = { foo = [(-1) 0 (-1)] -- negative values need parens in arrays bar = {"green": false} } -- newStruct = {foo = [(-1) 0 1]; bar = {"green": false, "blue": true}} newStruct = structure with { $.foo[2] = 1 $.bar["green"] = false $.bar["blue"] = true }

Note that updates are processed in order from top to bottom, which is relevant if a later update overrides an earlier update.

Via Update Functions

The Pointless standard library provides the following functions for adding elements to sets, and deleting elements and key / value pairs from sets and maps, respectively. Note that deleting a non-existing element or key from a set or make has no effect.

addElem({1, 2}, 3) -- {1, 2, 3} delElem({1, 2}, 2) -- {1} delElem({1, 2}, 0) -- {1, 2} delKey({Foo: "bar"}, Foo) -- {}

IO:

Notes on the input / output capabilities of Pointless.

Output Commands

Unlike other programming language, which use functions to send information from a program to the outside system, Pointless program communicate their output through a sequence of output commands. To accomplish this, Pointless designates a special variable, 'output', from which output commands are read when an program is run. Some programming languages have a main method, which serves as the entry-point for executing the program - in Pointless, the output variable also serves as an entry-point, where the value of the output variable is the output from the program.

Pointless expects the output variable to be defined as a list of commands. Commands can be either labels or labeled values - the two output commands which Pointless currently supports are 'IOPrint(value)', which directs the Pointless vm to print the string value, and 'IOClearConsole', which clears the console. The top-level source file for a Pointless program must define an output variable, and the value of output must be a valid list of commands - programs which do not meet these requirements will cause an error at runtime.

Since lists in Pointless are processed lazily, the sequence of output commands defined by a program will also be handled lazily. This means that, while programs must define a single entry-point for their output, that output is still handled as the programming running, rather than blocking until the program terminates.

IOPrint(value) IOClearConsole

To avoid interacting with these output commands directly, the Pointless standard library defines the following functions to help generate output sequences. The examples page demonstrates the many uses for these functions.

print(value) -- generate command sequence to print the string rep for value println(value) -- generate command sequence to print value with a newline printFrame(value) -- generate command sequence to clear console and print value with newline printLines(values) -- print each element in a sequence on a separate line

Because output sequences are made up of normal Pointless values, they themselves can be manipulated and visualized. In the example below, the code 'printLines(range(3))' generates a command sequence to print the numbers zero through two - the code then calls the 'println' function again to print a string representation of these commands. Running this code thus provides a visualization of a typical output command sequence.

output = printLines(range(3)) |> println

Input Commands

At present, Pointless provides two input sources - lines read from stdin, and random numbers. These inputs are accessible through special fields on the IO label, shown below.

IO.!getLine -- get the next line from stdin, or EOF if empty IO.!getRand -- get a random decimal value (between 0 and 1)

As with other special fields, the standard library has definitions for accessing these inputs: the 'readLines' variable, a lazy list of strings containing input lines from stdin, and several functions for generating random numbers / choices.

readLines -- read lines of input lazily randFloat(n) -- get random float in range(n) randRange(n) -- get random entry from range(n) randChoice(elems) -- get random elem from collection

While the set of input choices is currently limited, the existing input sources can be easily processed using existing functions. For example, the following code reads lines from stdin, and prints the length of each line.

output = readLines |> map(length) |> printLines

Imports:

Import statements are used to share code across multiple source files.

Import Statements

Import statements have the form 'import' "filePath" 'as' name, as shown below. This statement will load all of the definitions in the given source file into an object, which is then stored in the specified name. All import statements in a program must come before any variable or function definitions. Circular imports are allowed (two files can both import each other, for example), although this make the import objects themselves contain circular references.

import "chart.ptls" as chart

Assertions:

Pointless does not have specific handling errors after they occur - instead, functions should check for erroneous conditions before performing an operation that could lead to an error. For example, indexing a map with a key that the map does not contain will result in an error - this could be prevented by wrapping a map access in a conditional which check that the map contains the required key before attempting to index the map, and falling back to some other behavior if the key is not present.

if key in table then table[key] else (somethingElse)

Errors

There are many situations in which a Pointless program might terminate with an error. Some types of errors indicate a bug in the program - examples of these include:

  • Calling an operation with the wrong types of arguments (like adding two booleans)
  • Destructuring a tuple to the wrong number of members
  • Stack overflows caused by circular definitions or buggy recursive functions
  • Accessing a nonexistent field in an object

Additionally, there are cases where a correctly written program can fail in certain edge-cases. In these cases, errors can be avoided by checking values before an operation is performed, as in the map index example above. Examples of other errors which can be avoided using checks or refactoring include:

  • Division by zero
  • No matching case in a switch conditional
  • taking the head or tail of an empty list
  • Indexing an array with an out-of-bounds index
  • Indexing an array with a non-integer value

Requires

Requires statements are similar to assertions in other programming languages - they provide a way to check that a condition is met and throw an error if it is not. Unlike assertions, requires statements are attached to a specific expression, and take the form expression 'requires' condition, where the condition is checked before the expression is evaluated. This allows functions to verify preconditions before they run, as in the following function, which returns the last element in a list after checking that the list is not empty.

last(list) = head(reverse(list)) requires list is PtlsList

Prelude:

The prelude is the set of files which contain the definitions for the Pointless standard library. It currently includes 10 files (boolean.ptls, equals.ptls, format.ptls, function.ptls, io.ptls, list.ptls, numerical.ptls, set.ptls, show.ptls string.ptls) and about 900 lines of code.

Library Functions

The standard library defines the functions and variables below, which are available by default in all Pointless source files.

equals(a, b) -- preform deep equality check on two data structure notEquals(a, b) -- notEquals(a, b) = not equals(a, b) lessEq(b, a) -- lessEq(b, a) = a <= b lessThan(b, a) -- lessThan(b, a) = a < b greaterEq(b, a) -- greaterEq(b, a) = a >= b greaterThan(b, a) -- greaterThan(b, a) = a > b notFunc(x) -- notFunc(x) = not x notEq(a, b) -- notEq(a, b) = a != b eq(a, b) -- eq(a, b) = a == b orFunc(a, b) -- orFunc(a, b) = a or b andFunc(a, b) -- andFunc(a, b) = a and b inFunc(b, a) -- contains(b, a) = a in b any(values) -- takes a list of boolean values, returns true if any list value is true all(values) -- takes a list of boolean values, returns true if all list values are true compose(a, b) -- compose(a, b) = x => a(b(x)) const(a) -- const(a) = b => a iterate(func, init) -- get an infinte list [init, func(init), func(func(init)) ...] print(value) -- generate command sequence to print the string rep for value println(value) -- generate command sequence to print value with a newline printFrame(value) -- generate command sequence to clear console and print value with newline printLines -- print each element in a sequence on a separate line readLines -- read lines of input lazily randFloat(n) -- get random float in range(n) randRange(n) -- get random entry from range(n) randChoice(elems) -- get random elem from collection head(list) -- get the first element in a list tail(list) -- get all elements in a list after the first at(n, list) -- returns the nth element in a list (list must have at least n elements) slice(start, end, list) -- get a sublist of indices [start ... (end - 1)] concat(lists) -- lazily concatenate a list of lists into a single list concatMap(func, lists) -- map a list-generating function over a list and concatenate output lists intersperse(sep, list) -- make a new list with sep element interted between each pervious element repeat(elem) -- make an infinite list of a value, repeated take(n, list) -- get the first n elements in an list, or the whole list of length < n drop(n, list) -- get the elems after the first n elems in an list or empty if length < n takeWhile(func, list) -- take from a list the leading elements for which func returns true takeUntil(func, list) -- take elements up to (including) the first for which func returns true dropWhile(func, list) -- drop the leading elements for which func returns true dropUntil(func, list) -- drop elements up to (including) the first for which func returns true find(func, list) -- find the first element for which func return true, or None if none exists span(func, list) -- span(...) = (takewhile(...), dropWhile(...)) groupBy(func, list) -- return the list of lists of consecutive values for which func(a, b) == true suffixes(list) -- return all suffixes of a list, from longest to smallest map(func, collection) -- apply a function to each element in a list, and make a list of the results filter(func, list) -- apply a test function to each element, make new list of passing elements reduceFirst(func, list) -- reduce a non-empty list with first element set as accumulator reverse(list) -- get reverse of list toList(collection) -- convert a collection (a list, array, set, or tuple) to a list toArray(collection) -- convert a collection (a list, array, set, or tuple) to an array toSet(collection) -- convert a collection (a list, array, set, or tuple) to a set toMap(object) -- convert an object to a map of strings (field names) to values (field values) keys(mapVal) -- get a list of the keys in a map vals(mapVal) -- get a list of the values in a map items(mapVal) -- get (key, value) tuples for each entry in a map toNDArray(dims, elems) -- make an N-dimensional array from the values in a list length(collection) -- get the length of a collection hasPrefix(prefix, collection) -- return true if toList(collection) starts with the given prefix of elements enumerate(list) -- for a list [a, b, c, ...] return [(0, a), (1, b), (2, c) ...] sort(collection) -- sort a list of numbers sum(list) -- returns the sum of a list of numbers range(n) -- returns a list of numbers [0 ... (n - 1)] rangeStep(start, end, step) -- returns a list of numbers [start ... end] (inclusive) floor(n) -- round number down ceil(n) -- round number up toInt(val) -- convert a number or a string to an integer (truncates floats) toFloat(val) -- convert a number or a string to a float round(n) -- round a number to the nearest int value pi -- pi, to as many digits as I could remember euler -- e, to as many digits as I could remember abs(n) -- get the absolute-value of a number pow(b, a) -- pow(b, a) = a ** b mul(b, a) -- mul(b, a) = a * b div(b, a) -- div(b, a) = a / b mod(b, a) -- mod(b, a) = a % b add(b, a) -- add(b, a) = b + a sub(b, a) -- sub(b, a) = b - a max(a, b) -- returns the larger of two numbers min(a, b) -- returns the smaller of two numbers minimum(values) -- returns the smallest number in a non-empty collection maximum(values) -- returns the largest number in a non-empty collection repr(value) -- get the string rep of a value, keeping quotes if value is a string show(value) -- get the string representation of a value join(sep, collection) -- map a collection to strings, and join with a seperator string split(delimStr, string) -- return the list of substrings of a string, split by a delimiter padLeft(n, string) -- left-pad a string with spaces to make its length >= n padRight(n, string) -- right-pad a string with spaces to make its length >= n addElem(set, elem) -- add an element to a set delElem(set, elem) -- remove an element from a set delKey(mapVal, key) -- remove an entry from a map union(a, b) -- get the union of two sets intersection(a, b) -- get the intersection of two sets difference(a, b) -- get the difference of two sets symDifference(a, b) -- get the symmetric difference of two sets

Special Field Wrappers

Types in Pointless have a number of special files of the form '.!getFieldName', for example "123".!getInt (get the integer value of a string), {false}.!getAddElem (function to add an element to a set), and so on, which are used to expose built-in functions of Pointless types. The prelude contains wrapper functions that can be used to access this functionality, for example 'toInt' and 'addElem', which wrap access to the fields above.
